# Изменения: ссылка на маршрут Яндекс.Карт в маршрутах

Всё лежит незакоммиченным в рабочем дереве. Коммиты и пуш — за тобой.
Спека/план: `docs/superpowers/specs/2026-07-13-yandex-marshrut-ssylka-design.md`,
`docs/superpowers/plans/2026-07-13-yandex-marshrut-ssylka.md` (папка `docs` в .gitignore).

## Фронтенд — `c:\github\puteshestvuy_v_kchr`

- `src/lib/yandexRoute.js` (новый) — `parseYandexRoute(url)`: достаёт из URL Яндекс.Карт
  точки (`rtext`) и режим (`rtt`: pd→пешком, bc→вело, mt→транспорт, иначе auto).
- `src/components/YandexMapRoute/YandexMapRoute.jsx` (изменён) — два новых опциональных
  пропа: `routeOverride={{ points, mode }}` (строит маршрут по точкам/режиму из ссылки
  вместо координат мест; метки мест не меняются) и `openUrl` (кнопка «Открыть в
  Яндекс.Картах» ведёт на прикреплённую ссылку). Линия маршрута из ссылки рисуется
  даже если у маршрута нет добавленных мест (иначе карта показывала заглушку
  «Добавьте места…» и до ссылки не доходила). Точки маршрута из ссылки подписываются
  названиями через обратное геокодирование (когда у маршрута нет своих мест). Бейдж
  с длиной маршрута перенесён в центр линии (родной бейдж Яндекса скрыт через
  `routeActiveMarkerVisible: false`; координаты линии собираются через
  `getPaths()→getSegments()`). Без пропов поведение прежнее (кроме бейджа — он теперь
  в центре и для классических маршрутов). Проверено вживую на localhost.
- `src/sections/Routes/RouteDetail/RouteDetail.jsx` (изменён) — парсит `route.mapUrl`
  и прокидывает `routeOverride` + `openUrl` в карту «Как добраться».
- `src/app/admin/routes/[id]/page.jsx` (изменён) — в секции «Маршрут на карте» табы
  «Места на карте» / «По ссылке Яндекс.Карт»: первый показывает выбор мест (как раньше),
  второй — поле ссылки. Табы только переключают видимость, сохраняются оба поля
  (`mapUrl` и `placeIds`). При открытии маршрута с ссылкой активен таб «По ссылке».
  `mapUrl` добавлен в начальный стейт и в `getFormSnapshot` (dirty-трекинг). Превью
  «Карта маршрута» показывает маршрут из ссылки.

## Бэкенд — `C:\git back\puteshestvuy_v_kchr_back`

- `app/utils/yandexMap.js` (новый) — `resolveYandexMapUrl` / `normalizeYandexMapUrl`:
  разворачивает короткую ссылку `web-maps/-/…` в полный URL с `rtext` через системный
  `curl` (Node/axios Яндекс режет капчей — проверено). Только http(s) на `yandex.*`
  (защита от SSRF), best-effort (не роняет сохранение).
- `app/admin/routes.controller.js` (изменён) — в `createRoute`/`updateRoute` перед
  записью: `mapUrl = await normalizeYandexMapUrl(mapUrl)`.

## Проверено

- Реальная короткая ссылка `https://yandex.ru/web-maps/-/CHCYAMMr` разворачивается в URL
  с `rtext=…&rtt=pd`; edge-кейсы (пусто, не-Яндекс, `file://`) — ок.
- Парсер: `rtext`/`rtt` разбираются корректно, защита от prototype-ключей.
- Фронт собирается: `npm run build` — зелёный.

## Важное для деплоя

На сервере бэка должен быть установлен `curl` (обычно есть на Linux). Разворот короткой
ссылки делает один вызов `curl` (до ~8 c) в момент сохранения маршрута в админке.
