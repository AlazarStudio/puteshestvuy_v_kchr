# Дизайн: поддержка VK-плейлистов в поле «Аудиогид» мест

Дата: 2026-07-02
Статус: утверждён пользователем

## Задача

Сейчас поле «Аудиогид» у места принимает только ссылку Яндекс.Музыки (URL из атрибута `src` iframe-кода) и отображается как iframe в `PlaceModal`. Нужно, чтобы в это же поле можно было вставить код виджета VK-плейлиста вида:

```html
<div id="vk_playlist_-217757946_1"></div>
<script type="text/javascript" src="https://vk.com/js/api/openapi.js"></script>
<script type="text/javascript">
(function() {
VK.Widgets.Playlist("vk_playlist_-217757946_1", -217757946, 1, "", {});
}());
</script>
```

и плейлист отображался на странице места.

## Утверждённые решения

- **Оба источника**: поле принимает и Яндекс.Музыку (как сейчас), и VK-плейлист. Существующие места работают без изменений.
- **Официальный виджет VK**: рендер через `openapi.js` + `VK.Widgets.Playlist`, без недокументированных iframe-URL.
- **Только админка**: публичные формы «Предложить место» (`/suggest-place`, `SuggestPlaceModal`) не трогаем.

## Формат хранения

`audioGuide` остаётся одной строкой (бэкенд без изменений). Для VK вводится канонический токен:

```
vk_playlist:{ownerId}_{playlistId}_{hash}
```

Пример: `vk_playlist:-217757946_1_` (hash пустой). Ссылки Яндекс.Музыки хранятся как раньше (URL).

## Парсинг в админке

`handleAudioGuideChange` в `src/app/admin/places/[id]/page.jsx`. Порядок распознавания вставленного значения:

1. Код виджета VK — регуляркой достаём аргументы вызова
   `VK.Widgets.Playlist("...", <ownerId>, <playlistId>, "<hash>")` → в поле подставляется токен `vk_playlist:…`. Парсинг не зависит от переносов строк (при вставке в `<input type="text">` браузер их убирает).
2. Ссылка на плейлист `vk.com/music/playlist/{ownerId}_{playlistId}` (опционально `_{hash}`) → тот же токен.
3. Иначе — текущая логика: извлечение `src` из iframe-кода / URL Яндекс.Музыки.

UI-текст: лейбл — «Аудиогид (Яндекс.Музыка / VK)»; подсказка дополняется строкой про VK: «вставьте код виджета плейлиста VK (Поделиться → Экспортировать) — распознается автоматически».

## Отображение

В `src/components/PlaceModal/PlaceModal.jsx` (секция «Аудиогид»): если `place.audioGuide` начинается с `vk_playlist:` — рендерится новый компонент `VkPlaylistWidget`; иначе — существующий iframe Яндекс.Музыки без изменений.

## Новый компонент `VkPlaylistWidget`

`src/components/VkPlaylistWidget/VkPlaylistWidget.jsx`:

- принимает токен `vk_playlist:…`, парсит `ownerId / playlistId / hash`;
- загружает `https://vk.com/js/api/openapi.js` один раз на приложение (синглтон-промис на уровне модуля);
- после загрузки вызывает `VK.Widgets.Playlist(containerId, ownerId, playlistId, hash)` в контейнер с уникальным id (модульный счётчик — id из `useId` содержат `:`, что может сломать селекторы VK);
- при размонтировании очищает контейнер;
- при ошибке загрузки скрипта или нераспознанном токене ничего не рендерит — секция не ломает модалку.

## Проверка

Тестов в проекте нет. Ручная проверка через dev-сервер: вставка кода виджета в админке → значение превращается в токен → сохранение → виджет плейлиста отображается в модалке места; регресс Яндекс.Музыки — существующая ссылка отображается как раньше.
